执行 tide onboarding,目标是 10 分钟内让用户完成第一次有效分析。严格按顺序:

1. **验 key**:调用 `list_topics`(参数 `limit: 1`)。
   - 成功 → 告知「API key 有效,服务连通」。
   - 失败 → 按 `tide-intro` skill §8 转述错误 JSON 的 `reason` / `hint` / `portal_url`(常见:未配置 `TIDE_API_KEY` 环境变量、key 复制不完整、key 已吊销)。修复前不继续。
2. **跑一次真查询示范**:`search_topic(q="固态电池")`,把返回的三层候选(产业链/概念/题材)用一两句话讲给用户,证明链路可用。
3. **打印示例问题集**(告诉用户可以直接这样问):
   - 「我持有宁德时代和中际旭创,最近有什么我该知道的?」(/tide:portfolio)
   - 「今天开盘前我该知道什么?」(/tide:premarket)
   - 「300308 这只股票能不能买?」(/tide:stock)
   - 「固态电池现在还能追吗?」(/tide:theme)
   - 「人形机器人这条链卡在哪、谁最受益?」(/tide:chain)
   - 「英伟达这个新闻对 A 股什么影响?历史上类似事件后来怎么走?」(/tide:event)
   - 「今天涨停都是什么线索?复盘一下」(/tide:review)
   - 「帮我把持仓过一遍雷」「最近有什么新变化?」(/tide:portfolio 风险段、/tide:whatsnew)
4. **引导建持仓本地档**:问用户是否要建 `~/.tide/portfolio.md`(本地文件,不上云)。同意则创建,模板:

   ```markdown
   # 我的持仓
   | 代码 | 名称 | 成本 | 仓位 | 备注 |
   |---|---|---|---|---|
   | 300750 | 宁德时代 | 180.0 | 30% | 长线底仓 |
   # 关注(未持有)
   | 代码 | 名称 | 备注 |
   ```

   说明:云端自选(与 iOS App 同步)可用 `get_watchlist` / `set_watchlist`;若返回未上线提示,先用本地档。
5. 结尾附 `tide-intro` §6 免责框架,并提醒:报告为日更节奏(盘前 08:30 / 盘后 16:30 后),盘中突发可用 web search 补充。
